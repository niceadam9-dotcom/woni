### 11번 답변 (2026-07-10)

**Q1. 담당자 변경 시 갱신되는 페이지 명칭**

고객관리(고객 상세)와 지역별 담당 배정 어느 쪽에서 변경해도 같은 동기화 로직을 타며, 아래 화면이 모두 갱신됩니다:

| 갱신 화면 | 반영 내용 |
|---|---|
| 고객 관리 (목록·상세) | 고객의 담당직원 |
| 지역별 담당 배정 | 현 담당 표시 |
| 월간 점검계획 확정 | 미완료 계획 항목 + 진행중 점검에 연결된 항목의 담당 |
| 점검현황 모니터링 | 점검자 |
| 점검 업무 | 진행중 점검의 담당 |
| 점검 달력 | 담당자 뷰 분류 |
| (부가) 알림 | 새 담당자에게 배정 알림 (수신 설정 존중) |
| (부가) 활동 로그 | 변경 전→후 담당자 이력 |

**Q2. 고객관리 ↔ 지역별담당배정 상호 반영 여부 — 예, 서로 반영됩니다**

- 두 화면 모두 같은 데이터(customers.assigned_employee_id)를 읽고 쓰므로, 한쪽에서 변경하면
  다른 쪽에도 즉시 반영됩니다. 동기화 범위도 동일: 고객 담당 + 미완료 계획 + 진행중 점검
  + 진행중 점검 연결 항목(10번 보강분).
- **보강(이번에 적용)**: 화면 갱신(revalidate) 목록에서 서로의 화면이 일부 빠져 있던 것 보완 —
  고객관리 담당 변경 시 지역별 담당 배정 화면 갱신 추가, 지역별 일괄 배정 시 각 고객 상세 갱신 추가.
12. 담당자 변경과 같이 변경될 경우 다른 화면 변경되는 건들 무엇이 있는지 조사해줘
