## 상세

### 고객등록

#### ADD-1. 지역 필드 제거 (입력·추출·조회 삭제)
- **원문**: "지역 필드 없애주고, 추출도 없애줘, 지역조회안되게 삭제해줘"
- **구현 방향**: 등록 폼에서 `시/군/구 · 읍/면/동 · 리/동` 입력 3개 + 주소→지역 자동 추출 UI 제거
- **관련 파일**: `customer-new-client.tsx` (지역 입력 섹션, oncomplete의 region 세팅), `customers/new/page.tsx` (기본지역 prefill)
- ⚠️ **의존성 주의 — 착수 전 확인 필요**:
  - `region_si/myeon/ri` 데이터는 **지역별담당배정, 고객관리 읍/면 빠른 칩, 지역 필터, 검색(읍면/리)** 이 사용 중
  - 권장안: **화면에서만 숨기고, 주소검색 시 백그라운드 자동 저장은 유지** (지역 기능 유지)
  - 완전 삭제 시 위 기능들이 동작 불능 → 어느 쪽인지 결정 필요

#### ADD-2. 주소 중복 등록 감지 팝업
- **원문**: "주소가 이미 등록되어 있으면, 이미등록 되었다고 정보팝업, 팝업창 뜨면 관련 고객정보 조회"
- **구현 방향**:
  1. 주소검색 선택 시(또는 저장 시) 동일 주소(`customers.address`) 활성 고객 존재 여부 서버 조회
  2. 존재 시 팝업: "이미 등록된 주소입니다" + 해당 고객명/유형/담당자 표시 + [고객 상세 보기] 링크
  3. 등록 강행 허용 여부(중복 허용/차단) 결정 필요 — 기본: 경고 후 진행 허용
- **관련 파일**: `customer-new-client.tsx`, `customers/actions.ts` (주소 중복 조회 액션 신규)

#### ADD-3. 관계인 입력 — 대표만 기본, 나머지는 [추가] 버튼
- **원문**: "대표만 보이고 나머지 담당자, 직원2 삭제, 필요시 추가 버튼으로 추가"
- **현재**: 대표/직원1/직원2 3개 고정 카드
- **구현 방향**: 대표 1개만 기본 렌더 + [관계인 추가 +] 버튼으로 동적 행 추가(삭제 버튼 포함). 기존 V9 §9 '대표 1명 필수 + 추가 가능'과 일치
- **관련 파일**: `customer-new-client.tsx` (contacts state를 배열 구조로 변경), `customers/actions.ts` (ContactInput role 처리)

#### ADD-4. 🐛 주소검색 후 건물기본정보 자동 업데이트 안 됨
- **원문**: "건물기본정보가 주소검색 후 자동으로 업데이트가 안되고 있어"
- **추정 원인**: 등록 폼의 건물기본정보(용도/연면적/층수/준공)는 수동 입력 필드 — 주소검색(Daum)은 건물명만 추출하고 건물정보는 채우지 않음. 기대 동작 정의 필요: ① 같은 주소의 기존 buildings 데이터 자동 로드? ② Daum 데이터(공동주택 여부 등) 반영?
- **구현 방향(권장)**: 주소 선택 시 동일 주소 `buildings` 행 조회 → 있으면 용도/연면적/층수/준공 자동 채움 (ADD-2 중복 감지와 연계)
- **관련 파일**: `customer-new-client.tsx`, `customers/actions.ts`

#### ADD-5. 🐛 담당자 변경 시 점검이력 미기록
- **원문**: "점검이력에 담당자 변경시 변경이력이 없어, 강상로뎀병원"
- **재현 사례**: 강상로뎀병원 담당자 변경 → 이력 없음
- **점검 포인트**: `patchCustomerFieldAction`(인라인 수정)과 상세 페이지 '담당자 변경' 버튼 경로 각각의 이력 기록 여부 — 두 경로 중 한쪽이 누락 추정. 지역별담당배정 일괄 변경(bulkAssign) 경로도 확인
- **관련 파일**: `customers/actions.ts` (patchCustomerFieldAction, 담당자 변경 액션, bulkAssignEmployeeAction)

#### ADD-6. 🐛 고객 등록 시 점검이력 2건 중복 기록
- **원문**: "고객등록하면 왜 점검이력에 2건이 나오는 에러, 변경할 때만 변경내역을 기록"
- **추정 원인**: `createCustomerAction` 내에서 등록 기록 + 필드별 기록이 이중 삽입되거나, 등록 직후 내부 update가 '변경'으로 재기록
- **구현 방향**: 등록 시 이력 1건("고객 등록")만, 이후 실제 값 변경 시에만 변경 이력
- **관련 파일**: `customers/actions.ts` (createCustomerAction의 이력 기록 로직)

#### ADD-7. 고객등록 → 건물등록 주소·정보 자동 인계
- **원문**: "건물등록 버튼 누르고 들어가면 고객등록의 주소가 그대로 건물등록 주소에 자동저장, 관련 건물정보도 저장"
- **구현 방향**: 고객 상세(또는 등록 완료 화면)의 건물등록 진입 시 쿼리 파라미터/서버 조회로 고객 주소·우편번호·건물정보 pre-fill — "주소 2번 검색 불필요" (V9 고객등록 §2 연장)
- **관련 파일**: `buildings/new/page.tsx`, `building-new-client.tsx`, 진입 링크 위치
- **참고**: 고객 등록 시 건물 행 자동 생성은 이미 구현됨(A-10 PASS) — 이 항목은 별도 건물 '추가' 등록 경로의 인계
