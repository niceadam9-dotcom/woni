# 정산현황 모니터링 (회계연동) — 화면 설계 v1

> 점검계획이 생성된 건에 대하여 청구서 생성, 입금, 미납금 현황을 모니터링합니다.

---

## 1. 화면 개요

| 항목 | 내용 |
|------|------|
| 경로 | `/billing/status` |
| 접근 권한 | manager / admin |
| 주요 기능 | 월별 청구서 목록 조회, 입금 처리, 미납금 추적, 세금계산서 발행 현황 연동 |
| 하단 액션 | [청구등록] 버튼으로 신규 청구서 등록 |

---

| 필터 | 옵션 | 설명 |
|------|------|------|
| 상태 드롭다운 | 전체 / 대기 / 입금완료 / 미납 | 청구서 수납 상태 |
| 청구일 범위 | DATE ~ DATE | 청구서 발행 예정일 기준 기간 |

## 4. 컬럼 상세 정의

| 컬럼 | 설명 | 데이터 소스 | 비고 |
|------|------|-------------|------|
| No | 행 번호 | — | |
| 청구월 | 청구 귀속 연월 (YYYY.MM) | `bills.billing_month` | |
| 구분 | 청구 구분 (일괄점검 / 자동기능점검 / 일괄청구 등) | `bills.bill_type` | |
| 건물명 | 고객(건물)명 | `customers.customer_name` | |
| 청구일 | 청구서 발행 예정일 또는 발행일 | `bills.bill_date` | |
| 입금일 | 실제 수납 완료일 | `bills.paid_at` | 입력 전 공란 |
| 공급가액 | 부가세 제외 금액 | `bills.supply_value` | |
| 부가세액 | 부가세 10% | `bills.tax_value` | |
| 청구금액 | 공급가액 + 부가세액 | `bills.total_amount` | |
| 미납금액 | 청구금액 − 입금액 (미수금) | 계산값 | 입금 완료 시 0 |
| 계산서발행 | 세금계산서 발행 완료 여부 (✓ / —) | `tax_invoices.issued` | |
| 선택 | 일괄 처리용 체크박스 | — | |

## 5. 행 표시 규칙

| 상태 | 조건 | 표시 |
|------|------|------|
| 입금완료 | `paid_at IS NOT NULL` AND `미납금액 = 0` | 정상 (흰 배경) |
| 입금대기 | `paid_at IS NULL` | 기본 표시 |
| 특별관리 | 관리자가 별도 플래그 설정한 행 | 회색 배경 (`bg-gray-50`) |
| 미납 강조 | `청구일 + 납기일 < today` AND `paid_at IS NULL` | 미납금액 셀 `bg-red-100 text-red-700` |

## 6. [청구등록] 버튼 — 신규 청구서 등록 모달

```
┌─────────────────────────────────────────────────┐
│  청구서 등록                              [×]   │
├─────────────────────────────────────────────────┤
│  건물명 *      [검색                      🔍]   │
│  청구월 *      [2025.04                    ]    │
│  구분   *      [일괄점검              ▼]        │
│  청구일 *      [2025-04-11             📅]      │
```

- 공급가액 입력 시 부가세액(×10%), 청구금액 자동 계산
- 건물명 선택 시 해당 점검계획 자동 연결

## 7. 입금 처리 — 행 클릭 슬라이드 패널

```
┌──────────────────────────────────────────┐
│  을지초등학교 — 2025년 4월 청구    [×]  │
├──────────────────────────────────────────┤
│  청구월       2025.04                    │
│  구분         일괄점검                   │
│  청구금액     110,000원                  │
│                                          │
│  입금일  *    [2025-04-01      📅]  [×] │
│  입금액  *    [         110,000   ]     │
│  입금방법     [계좌이체          ▼]     │
│  메모         [                   ]     │
│                                          │
│  [저장]                                  │
└──────────────────────────────────────────┘
```

- 입금일 + 입금액 저장 → `bills.paid_at`, `bills.paid_amount` 업데이트
- 미납금액 자동 재계산
- 세금계산서 발행 연동 버튼 활성화 (입금 완료 후)

## 8. 합계 행

| 컬럼 | 내용 |
|------|------|
| 공급가액 합계 | 조회된 전체 건 공급가액 SUM |
| 부가세액 합계 | 조회된 전체 건 부가세액 SUM |
| 청구금액 합계 | 공급가액 합계 + 부가세액 합계 |
| 미납금액 합계 | 미납 건 미납금액 SUM (0 = 전액 수납 완료) |

---

## 9. 하단 범례

> ★ **청구후시**: 세금계산서 발행 당사자가 아닌 경우, 청구 금 기준 세금계산서 형식으로 환급합니다.  
> ★ **청구후시**: 세금계산서 발행 당사자에게 당일 '세금계산서 형식'으로 발행 후, 세금계산서 내용에 따른 입금방법을 등록합니다.

## 10. DB 테이블 설계

### 10-1. bills (청구서)

| 컬럼 | 타입 | 비고 |
|------|------|------|
| id | UUID PK | |
| customer_id | UUID → customers(id) | |
| inspection_plan_item_id | UUID → inspection_plan_items(id) | 점검계획 연결 (선택) |
| billing_month | VARCHAR(7) | 'YYYY.MM' |
| bill_type | VARCHAR(30) | 일괄점검 / 자동기능점검 / 일괄청구 등 |
| bill_date | DATE | 청구서 발행(예정)일 |

| created_at / updated_at | TIMESTAMPTZ | |

> `미납금액 = total_amount - paid_amount` — 앱에서 계산 (또는 GENERATED 컬럼)

## 11. 비즈니스 규칙

| 규칙 | 내용 |
|------|------|
| 청구 자동 생성 | 점검계획 확정 시 청구서 초안 자동 생성 (bill_type = 점검 유형 반영) |
| 부가세 자동 계산 | 공급가액 입력 시 `tax_value = supply_value × 0.1` 자동 산출 |
| 미납금 계산 | `미납금 = total_amount - paid_amount`, `paid_amount` 미입력 시 `total_amount` 전액 미납 |
| 미납 강조 | 납기일 초과 & `paid_at IS NULL` → 미납금액 셀 빨간 강조 |
| 세금계산서 연동 | 입금 완료 후 세금계산서 발행 버튼 활성화, 발행 완료 시 `issued = true` |
| 합계 행 | 필터 조건 적용된 조회 결과 전체 합산 (페이지 단위 아님) |
| 청구등록 권한 | manager / admin 전용 |

## 12. 상태 흐름도

```
[점검계획 확정]
      │
      ▼
[청구서 초안 자동 생성]
  bill_date 설정, paid_at = NULL
      │
      ▼
[청구 발송]  ──→  관계인에게 청구서 전달
      │
      ▼
[입금 대기]
      │
      ├─ 입금 확인 ──→ paid_at, paid_amount 기록
      │                     │
      │                     ▼
      │               [입금완료] ──→ 세금계산서 발행 활성화
      │                                  │
      │                                  ▼
      │                           [세금계산서 발행완료]
      │
      └─ 납기 초과 ──→ 미납금 빨간 강조, 담당자 알림
```

## 15. 상단 KPI 카드 (4개)

```
┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
│  🔴              │  │  🔵              │  │  🟢              │  │  🟣              │
│  5,224,548       │  │        0         │  │  1,150,000       │  │        0         │
│  금년 매출누계   │  │  금월 미출고누계 │  │  금년 미납누계   │  │  금월 수주누계   │
└──────────────────┘  └──────────────────┘  └──────────────────┘  └──────────────────┘
```

| 카드 | 지표 | 데이터 소스 | 집계 기준 |
|------|------|-------------|----------|
| 금년 매출누계 | 당해연도 청구금액 합계 | `bills.total_amount` SUM WHERE 연도 = 올해 | 발행 기준 |
| 금월 미출고누계 | 이달 미처리 출고 건 수 | `bills` WHERE `paid_at IS NULL` AND 청구월 = 이달 | 미납 건수 |
| 금년 미납누계 | 당해연도 미납금 합계 | `bills.total_amount - paid_amount` SUM | 금액 기준 |
| 금월 수주누계 | 이달 신규 수주 금액 합계 | `orders.amount` SUM WHERE 월 = 이달 (영업모듈) | 수주 기준 |

## 19. 모바일 앱 연동

이미지에 스마트폰 화면이 함께 표시된 것처럼, 대시보드는 모바일 앱과 동일한 데이터 소스를 공유합니다.

| 항목 | 내용 |
|------|------|
| 접근 방식 | 반응형 웹 (Tailwind breakpoints) + 별도 모바일 앱 |
| 공유 데이터 | Supabase Realtime 구독으로 KPI 수치 실시간 반영 |
| 알림 | Supabase `notifications` 테이블 → 앱 푸시 알림 연동 |
