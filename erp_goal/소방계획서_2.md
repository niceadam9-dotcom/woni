## A. 표준양식 생성(웹 모달) — 버그·데이터 연동 수정

## B. 개정이력 연동 (항목 0·1·2)

현재 템플릿은 개정이력 표에 **이번 생성분 1행만** 출력한다. 요구사항:

- **0**: 이미 저장된 소방계획서(같은 고객)가 있으면 다음 순번에 일자·개정내용이 **이어서** 누적 출력
- **1**: 개정이력 "일자" = 표준양식 생성 폼의 "작성일"(revisionDate)
- **2**: 개정이력 "주요 개정내용" = 생성 폼의 "개정 내용"(revisionNote)

수정 방향: 생성 시 같은 고객의 기존 `fire_plans` 행들(revision·revision_note·작성일)을 조회해
개정이력 표에 순번대로 누적 렌더링 + 이번 생성분을 마지막 행으로 추가. 🟡

## E. 화면 구조 개편 (항목 14·23)

- **14**: "표준양식 생성"을 고객관리의 **탭**으로 승격
- **23**: 종합 요구 —
  1. 한글 소방계획서 문서 기반의 모든 입력란을 **고객관리 > 소방계획서 탭**에서 입력
  2. 소방시설현황(현 별도 탭/패널)을 **소방계획서 탭 아래로 이동**
  3. 진입 방식을 "표준양식 생성 버튼 → 모달"이 아니라 **소방계획서 탭에서 직접 입력**하는 구조로 변경

방향(제안): 소방계획서 탭을 서식 목차(1.1 일반현황 / 1.4 시설 / 1.5 피난·방화 / 1.6 기타 / 1.7 자위소방대 / 1.10 점검 / 1.11 훈련) 그대로의
섹션 구조로 재편 → 입력 즉시 저장 → "생성" 버튼은 탭 상단에서 HWP/PDF 출력만 담당. D 묶음의 기반 작업이므로 **D보다 먼저** 골격을 잡는 것이 효율적.
