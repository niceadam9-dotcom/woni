# 승진소방 점검 업무 관리 모듈 — 목표 문서

## 모듈 개요

소방 점검 업무의 전 주기(고객 등록 → 점검 배정 → 7단계 워크플로우 → 보고서 제출)를
웹 ERP에서 관리하고, **Microsoft Outlook 달력과 동일한 UX**로 담당자별·고객별 일정을 시각화한다.

기존 ERP(기안서·휴가)와 동일한 기술 스택(Next.js 16 / Supabase / react-big-calendar) 위에 구축한다.

---

## 핵심 기능

### 1. 고객 관리
- 고객 등록: 고객코드(UNIQUE), 고객명, 계약일, 점검유형(종합/최초/기타), 주소
- 관계인 등록: 고객당 최대 3명(대표·직원1·직원2), 이름·연락처·이메일
- 점검유형별 연간 점검 횟수 자동 적용
  - 종합: 연 2회 (1차·2차)
  - 최초: 연 1회
  - 기타: 연 1회

### 2. 점검 업무 등록 및 배정

- 점검 생성: 고객·관계인·담당직원·점검시작일 지정
- **점검 생성 즉시 7단계 업무와 마감일이 자동 계산되어 생성됨**
- 업무할당: 승진소방 담당자 배정 → 해당 직원 달력에 자동 반영
- 고객 관계인과의 협의 일정도 달력에 표시

### 3. 7단계 워크플로우 — 자동 마감일 계산

점검시작일 기준, 공휴일(holidays 테이블) 반영하여 자동 계산.

| 단계 | 업무명 | 기준일 | 기한 | 계산방식 |
|---|---|---|---|---|
| 1 | 관계인 통보 1차 | 점검시작일 | 3일 | 작업일 (토·일·공휴일 제외) |
| 2 | 관계인 협의 2차 보강 | 점검시작일 | 4일 | 작업일 |
| 3 | 보고서 작성 | 점검시작일 | 9일 | 작업일 |
| 4 | 소방서 제출 | 3단계 마감일 | 1일 | 작업일 |
| 5 | 공사완료 | 점검시작일 | 10일 | 달력일 (공휴일 포함) |
| 6 | 이해관계자 보고서 만들다 | — | — | 마감일 없음 |
| 7 | 이행완료 보고서 제출 | — | — | 마감일 없음 |

각 단계는 담당 직원이 완료 체크 가능. 완료 시 완료일시 및 완료자 기록.

### 4. Outlook 달력 계획표 (핵심 기능) ★

**`/inspections/calendar`** — Microsoft Outlook 달력과 동일한 UX로 모든 점검 일정 시각화.

#### 4-1. 담당자 뷰 (승진소방 직원 기준)
- 각 직원의 점검 단계 마감일을 달력 이벤트로 표시
- 직원별 색상 구분 (10색, 기존 팀 휴가 달력 패턴 재사용)
- 완료된 단계: 반투명 + 체크 배지
- 기한 초과: 빨간 테두리 강조

#### 4-2. 고객 뷰 (고객 관계인 기준)
- 동일한 달력에서 고객별 색상으로 이벤트 표시
- 고객이 수신·처리해야 하는 단계 강조 (1차 통보 수신, 2차 협의)
- 고객명 검색 + 체크박스 필터

#### 4-3. 공통 기능
- **뷰 전환**: 월간(Month) / 주간(Week) / 일(day)/목록(Agenda)
- **왼쪽 필터 패널** (Outlook 사이드바 스타일):
  - 뷰 토글: 담당자 뷰 ↔ 고객 뷰
  - 직원 선택 (색상 스와치 + 체크박스)
  - 고객 검색 + 체크박스
  - 점검유형 필터: 종합 / 최초 / 기타
  - 상태 필터: 미완료 / 완료 / 기한초과
- **이벤트 클릭**: 해당 점검의 상세 패널 열림
  - 고객명, 담당자, 점검유형, 시작일 표시
  - 7단계 진행률 바
  - 각 단계: 마감일 + 상태 배지 + 완료 체크 버튼 (본인 담당 점검만 활성)

### 5. 보고서 관리

보고서 3종, 점검당 유형별 1건:

| 보고서 유형 | 설명 |
|---|---|
| 점검 보고서 (check_station) | 소방서에 제출하는 공식 보고서 |
| 소방서 제출용 (fire_station) | 소방서에 제출하는 공식 보고서 |
| 이해관계자 보고서 (stakeholder) | 건물주·관계인용 보고서 |
| 이행완료 보고서 (completion) | 공사완료 및 이행 확인 보고서 |

각 보고서: 파일 첨부(Supabase Storage), 제출일시·제출자 기록.
고객코드·고객명은 제출 시점의 스냅샷으로 저장 (이후 고객정보 변경과 무관).

### 6. 공휴일 관리 — 국가공휴일 자동 동기화 ★

작업일 계산(`add_working_days`)은 `holidays` 테이블을 참조한다.
공휴일이 정확해야 점검 마감일이 올바르게 계산되므로 매년 초 동기화가 필요하다.

#### 국가공휴일 자동 동기화
- `date-holidays('KR')` 패키지로 연도별 한국 공휴일 전체를 fetch
- admin 페이지에서 연도 선택 후 **"동기화" 버튼 1회 클릭**으로 완료
- `UPSERT (onConflict: 'date')` — 이미 등록된 날짜는 이름만 갱신, 중복 추가 없음
- 지원 연도: 전년도 / 당해연도 / 내년도 (3개년 선택 가능)

#### 회사 자체 휴무일 수동 등록
- 날짜 + 이름 입력으로 추가 (예: 창립기념일, 임시공휴일)
- `is_national = false` 로 국가공휴일과 구분
- 목록에서 개별 삭제 가능

#### 화면 구성 (`/admin/holidays`)
| 영역 | 기능 |
|---|---|
| 국가공휴일 자동 동기화 카드 | 연도 선택 + 동기화 버튼, 결과 메시지 |
| 회사 자체 휴무일 추가 카드 | 날짜 input + 이름 input + 추가 버튼 |
| 등록 공휴일 목록 | 날짜·요일·이름·유형 배지, 삭제 버튼 |

## 데이터베이스 설계 요약

### 신규 테이블 6개

| 테이블 | 설명 |
|---|---|
| `holidays` | 공휴일 (작업일 계산용) |
| `customers` | 고객 (고객코드 UNIQUE, 점검유형, 계약일) |
| `customer_contacts` | 관계인 (고객당 대표·직원1·직원2) |
| `inspections` | 점검 업무 (고객+직원 배정, 점검시작일) |
| `inspection_steps` | 7단계 업무 (점검 INSERT 시 자동 생성) |
| `inspection_reports` | 보고서 3종 (파일 첨부, 스냅샷) |

### DB 자동화 로직

- `add_working_days(date, n)` — holidays 테이블 기반 작업일 계산 함수
- `create_inspection_steps()` — 점검 생성 시 7단계 자동 생성 트리거
- `check_inspection_sequence()` — 종합 외 2차 점검 생성 방지 트리거

### RLS (Row Level Security)

- 배정된 직원: 본인 담당 점검·단계만 조회/수정
- manager/admin: 전체 조회, 고객·점검 생성·삭제
