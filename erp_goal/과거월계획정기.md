# 과거월 계획 정기 — 규칙·사건 기록 (2026-07-10)

연간 점검계획의 **정기(자체점검, monthly)** 항목이 이미 지난 달에 생성·잔존해 달력에 지연(⚠)으로 표시되던 문제의 원인, 조치, 확정 규칙 문서.

## 현상

- 점검달력에 `샤브올데이·정기 ⚠` (6/29, 지연) 칩이 버그 수정 후에도 계속 표시됨 (image-17.png로 보고됨)
- 샤브올데이·탑텐의 2026년 1~6월 정기 항목 12건이 planned 상태로 잔존

## 원인 — 순서 문제 + 수정 범위

```
7/9 13:31  샤브올데이 재등록(취소 후 재등록) → 등록 직후 연간계획 자동 생성
           └─ 이 시점 코드는 수정 전: 과거 월(1~6월) 정기까지 전부 생성
7/10 00:22  "지난달 정기 생성 생략" 수정 커밋 (e2569a6) — 재등록보다 11시간 늦음
```

1. **순서**: 재등록 시 자동 생성이 버그 수정보다 먼저 실행됨
2. **수정 범위**: e2569a6는 생성기 로직만 변경 — 이미 저장된 행은 삭제하지 않음
3. 7/6 점검 시작·완료는 7월 특별(작동) 항목과만 연결 — 1~6월 정기와 무관

## 확정 규칙 (설계)

1. **기준일** = 최초 점검시작일 우선 → 없으면 사용승인일 (`loadAnchorDates` 단일 소스)
2. **지난 달 정기는 생성하지 않음** — 수행 불가한 과거 유령 항목 방지 (중도 등록·재등록·올해 보정 대응)
3. **기준일 이전 날짜의 항목은 생성하지 않음** — 최초 점검 전에는 이행 의무 없음
4. **특별점검(종합/작동)은 과거 월이어도 생성** — 법정 의무, 미점검 초과 해결 플로우 대상 (단, 규칙 3에 걸리는 경우 제외)
5. **특별월에는 정기 없음** — 연 12건 구성 (사용자 확정, 2026-07-09)
6. **미이행 정기의 월말 자동 처리는 하지 않음** — 지연 표시를 유지해 관리자가 인지·수동 처리 (현행 유지)
