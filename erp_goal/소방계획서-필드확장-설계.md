# 소방계획서 ERP 연계 설계 — 차수별 검토 + 5·6·7차 계획 (2026-07-15 개정 2판)

## 1. 목적

표준양식(53쪽)을 ERP 데이터로 단계 연계한다. 1~4차 완료. 이 문서는 그 검토와 함께
**5·6차(DB 필드 확장) + 7차(공통 프리셋)** 설계, 그리고 **사용자가 쉽게 입력·확인하는 UX**를 정의한다.

## 4. 입력 UX 설계 — 그룹화 (사용자가 쉽게 입력)

**위치: 고객 상세 → 소방계획서 카드 → [계획서 정보] 아코디언 패널** (새 화면 없음)

```
[소방계획서]  계획서 준비율 ▓▓▓▓▓▓░░ 14/18   [계획서 정보 ▾]
├─ 🏢 건물 개요        수신기위치 [1층 관리실▾] 구조 [철근콘크리트▾] 지붕 [슬래브▾]
│                     급수  ( 특급 | 1급 | 2급 | ●3급 )  ← 세그먼트 버튼(클릭 1회)
├─ 🛡 화재보험         가입 (●예|아니오)  보험사[    ] 기간[    ] 대인[  ]원 대물[  ]원
│                     └ '아니오'면 나머지 입력칸 자동 숨김
├─ ⏰ 운영·인원        평일[09~18▾] 휴일[미운영▾]  근무[ ]명 거주[ ]명 최대수용[ ]명
├─ 🚒 자위소방대       [행: 구분▾|성명|임무(기본문구 자동)|연락처]  [+ 행]  ← 관계인에서 가져오기 버튼
└─                                                      [저장]
```

입력을 쉽게 만드는 장치 4가지:
1. **선택식 우선** — 구조·지붕·운영시간은 **드롭다운 프리셋** + 직접입력 허용. 급수·보험 가입은 **세그먼트/토글 버튼**(타이핑 0회)
   - 구조: 철근콘크리트/철골/조적/목구조/샌드위치판넬 · 지붕: 슬래브/기와/판넬/징크
   - 운영시간(확정): **24시간 / 09~18시 / 주간만 / 미운영 / 직접입력**
2. **자동 제안** — 건축물대장 연동값(구조)·건물 용도 기반 기본값을 미리 채워 놓고 확인만
3. **가져오기 버튼** — 자위소방대 행에 [가져오기]: **관계인 + 승진소방 직원(profiles) 둘 다 후보 목록**으로 표시, 선택 시 대장/부대장/반원으로 원클릭 추가, 임무는 표준 문구 자동 (2026-07-15 확정: 업무대행 특성상 직원도 편성 대상)
4. **그룹별 저장 불필요** — 패널 하단 [저장] 한 번에 전체 저장 (그룹은 시각적 구분만)

## 5. 확인 UX 설계 — 쉽게 체크 (누락을 한눈에)

1. **계획서 준비율 게이지** — 카드 상단 `▓▓▓░ 14/18` : 연계 대상 필드 중 입력된 수. 클릭하면 **누락 항목 칩** 펼침:
   `누락: [지붕] [화재보험] [자위소방대]` — **칩 클릭 시 해당 입력칸으로 스크롤·포커스**
2. **생성 페이지 사전 체크** — 고객 선택 시 준비율과 누락 칩을 요청 버튼 옆에 표시 → "이대로 생성"(빈 칸 허용) 또는 "입력 후 생성" 선택
3. **생성 후 체크** — (기존 3차 기능) 결과에 누락 목록 + 고객 상세 링크 → 입력 → 재생성 순환

## 7. 7차 — 공통 수기 프리셋 (건물 유형별) ✅ 구현 완료 (2026-07-15)

- 훈련 시나리오·피난 절차 등 수기 문구를 `fire-plans/_presets/{house|retail|factory}.json`(주택형/상가형/공장형)으로 관리
  - **파일명 ASCII 매핑**: Supabase Storage가 한글 키를 거부(InvalidKey) — 매핑은 `src/lib/fire-plan-presets.ts`의 `PRESET_FILE_KEYS`
  - 스키마: `entries[] = {section, title, find(양식 기본값), value(프리셋 문구)}` — value가 비었거나 find와 같으면 양식 기본값 유지
  - 기법: 양식에 미리 채워진 예시 수기 문구를 앵커로 **단순 치환** (긴 문구 우선 정렬 — '1층 주차장' 같은 짧은 앵커가 긴 문구를 먼저 깨뜨리지 않게), 앵커 12종
- 생성 시 프리셋 선택(용도 기반 자동 추천 `recommendPresetType`) → 문구 주입. 우선순위: **고객 필드 > 프리셋 > 양식 기본값**
  - 프리셋 치환은 최저 우선순위로 병합(make-fireplan.py `generate_hwp(preset_pairs=)`), 파일 없음·손상은 fail-soft(양식 기본값 유지)
  - 요청 시 프리셋 파일이 없으면 기본값으로 자동 시딩(`ensurePresetFile`) — CLI는 `--preset 주택형|상가형|공장형`
- 일괄 생성: 프리셋 공유 고객 다중 선택(칩 UI, 최대 30건) → 워커 순차 처리 (연초 일괄 발행) — `requestFirePlanHwpAction(customerIds[])`
- 프리셋 관리 UI: `/fire-plans/generate` 하단 [공통 수기 프리셋 관리] 아코디언 — 유형 탭·문구 편집·기본값 복원·저장
- 자유 수기는 [연차발행](전년 복제)으로 보존 — 재입력 없음
- 검증(2026-07-15): 케이엔몰(상가형) CLI E2E — 문구 13회 치환·양식 기본값 잔존 0·고객 필드 우선 확인, tsc/eslint/py_compile 통과
