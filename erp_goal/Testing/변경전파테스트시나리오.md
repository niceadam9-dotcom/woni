# 변경 전파 테스트 시나리오 (TS-PROP)

작성: 2026-07-10. 변경전파맵.md/.json의 PROP-1~20 각각에 대한 상세 테스트 시나리오.
추적용 상태는 변경전파맵테스트.json 참조 (기존 자동 테스트로 커버되는 12건 = passed, 미검증 8건 = pending).

**공통 원칙**
- 테스트 데이터는 `TEST-` 접두 임시 고객·계정으로 만들고 종료 시 전부 정리 (운영 DB 공유 주의 — mock 시딩 금지)
- 자동화 시 기존 패턴 재사용: 임시 admin 계정 생성 → Playwright 로그인 → UI 구동 → DB 교차검증 → 정리
- 상태 확인은 반드시 "반영 화면" 전부에서 — 한 화면만 보면 10번(탑텐) 같은 부분 불일치를 놓침

## 2. 업무(전자결재·휴가)

### TS-PROP-11. 결재 상신→승인/반려 전파 ⏳ 미검증
- **사전조건**: 임시 계정 3개 — 기안자 A(employee), 결재자 B(manager), 결재자 C(admin)
- **단계**: ① A: 기안서 작성·상신(결재선 B→C) ② B: 결재함에 도착 + 알림 수신(필수 — 설정 꺼도 수신)
  ③ B 승인 → C 결재함 도착 ④ C 최종 승인 → 문서 상태 approved, A에게 결과 알림
  ⑤ **프로브 1**: A가 결재 결과 알림을 끄고(설정) 반려받으면 → 문서함 상태는 반려로 갱신되되 알림만 미수신
  ⑥ **프로브 2**: 관리자 문서관리에서 상태 일치 확인
- **자동화**: 브라우저 패턴으로 가능 (3계정 컨텍스트)

### TS-PROP-12. 휴가 신청→승인 전파 ⏳ 미검증
- **사전조건**: 임시 직원 A(연차 15일), 팀장 B, 관리자 C
- **단계**: ① A: 휴가 신청(연차 2일) ② B(승인자) 알림 수신(필수) + 목록 노출 ③ B 승인 → C 최종 승인
  ④ A 연차 잔여 15→13 (직원관리·설정 내 정보에서 확인) ⑤ 휴가 달력에 A 휴가 표시
  ⑥ **프로브**: 반려 시 연차 차감 없음 + 결과 알림 수신 설정 존중
- **자동화**: 브라우저 패턴으로 가능
