# 변경 전파 맵 — 한 곳 변경 → 여러 화면 반영 전수 목록

작성: 2026-07-10 (수정사항리스트 12번 조사 결과). 시스템에서 하나의 변경 행위가 어떤 데이터를 함께
바꾸고 어떤 화면에 반영되는지의 전체 지도. 화면 갱신 문제(예: "바꿨는데 다른 화면엔 옛 값")를
조사할 때 이 문서에서 해당 행위를 찾아 반영 범위를 확인한다.

## 공통 원리

- 전파는 **서버 액션의 동기화 로직**(연쇄 데이터 변경) + **revalidatePath**(화면 갱신)로 구현
- 2026-07-10 보강으로 점검 계열 액션의 갱신 범위 통일 (수정사항리스트 3·10·11번)
- 데이터 정합 상시 검사: `node scripts/check-plan-invariants.mjs` (INV-P1~P6)

## 1. 소방안전관리 계열

### 1-1. 고객 등록 — 고객관리 > 신규 등록
- **함께 생성**: 연간 점검계획 자동 생성(특별+정기, 12건/년,
  기준일=점검계획일(필수 수동 입력, 최우선)→최초 점검시작일→사용승인일),
  건물 자동 등록, 담당 지정 시 배정 알림
- **반영 화면**: 점검확정(월간 점검계획), 점검달력, 건물관리
- **구현**: customers/actions.ts `createCustomerAction` → `_autoCreatePlanItemsForNewCustomer`
- **표시 규칙(2026-07-14)**: 점검확정 화면의 "점검계획일" 컬럼은 plan_anchor_date 원본 표시
  (미입력 고객은 빨간 "미입력" + 인라인 입력 유도). 날짜 자동 제안·배분은 기준일(anchor_date) 사용.
  사용승인일 라벨은 고객관리·점검대장의 원본 값에만 사용

### 1-2. 담당자 변경 — 고객관리 상세 / 지역별 담당 배정 / 직원관리 인수인계
- **함께 변경**: 고객 담당 + 미완료 계획 항목 + 진행중 점검 + 진행중 점검에 연결된 항목
- **반영 화면**: 고객관리(목록·상세), 지역별 담당 배정, 점검확정, 점검현황 모니터링, 점검업무, 점검달력
- **부가**: 새 담당자 알림(수신 설정 존중), 활동 로그(변경 전→후)
- **구현**: `assignEmployeeAction` / `bulkAssignEmployeeAction` → `_syncEmployeeToRelated`

### 1-3. 고객 비활성 ↔ 재활성 — 고객관리 목록 토글
- **함께 변경**: 미완료 계획 자동취소(⟦자동취소:원상태⟧ 마커 보존) ↔ 재활성 시 원상태 복원.
  완료·기존 수동취소 항목은 불변
- **반영 화면**: 고객관리, 점검확정(취소 칩), 점검달력(취소선+비활성 배지), 미점검 초과 경보
- **구현**: `toggleCustomerActiveAction` → `_autoCancelPlansForCustomer` / `_restorePlansForCustomer` (ADD-16)

### 1-4. 고객 삭제(소프트) — 고객관리
- **함께 변경**: 비활성 전환 + 미완료 계획 자동취소 (1-3과 동일 메커니즘, 복원은 재활성으로)
- **반영 화면**: 고객관리, 건물관리, 점검확정, 점검달력
- **구현**: `deleteCustomerAction`

### 1-5. 점검계획일·사용승인일(기준일) 변경 — 고객 수정 폼 / 고객관리 인라인 편집 / 점검확정 인라인 입력
- **함께 변경**: 미확정(planned) 항목만 예정일 재계산 + 확정상태·6단계 일정 초기화.
  확정(confirmed)·완료·취소 항목은 불변 (2026-07-12 결정)
- **확정 일정 보유 시(2026-07-14, B안)**: 저장 전 팝업으로 확정 항목 목록 표시 →
  ①확정해지 후 전체 재계산(점검 미시작 건만 planned 복귀) ②확정 유지(미확정만 재계산) ③취소.
  선택 전에는 아무것도 저장되지 않음
- **점검계획일 비우기 불가**(2026-07-14): 필수값 — "지우면 폴백 복귀" 설계 폐기.
  미입력 기존 고객(필수화 이전 등록)은 화면에서 "미입력" 표시로 입력 유도
- **반영 화면**: 점검확정, 점검달력, 고객 상세, 고객관리 목록
- **구현**: `updateCustomerAction` / `patchCustomerFieldAction` → `_getUnconfirmablePlanItems` + `_resetPlanItemsForCustomer`

### 1-5b. 점검유형 변경 — 고객 상세 모달 / 고객관리 인라인 편집
- **함께 변경**: 고객 유형·category·sub_type + 미확정(planned) 항목의
  inspection_type·sub_type·plan_type(special_종합↔special_작동) 동기화.
  작동 전환 시 미확정 2차 특별점검 삭제, 종합 전환 시 연간 항목 보충 생성(멱등, 2차 포함),
  일반관리 전환 시 소방안전관리 자동 계획(planned) 삭제. 확정·완료·취소 항목은 불변
- **반영 화면**: 고객관리(목록·상세), 점검확정, 점검달력
- **구현**: `updateCustomerAction` / `patchCustomerFieldAction` → `_syncInspectionTypeToPlanItems` (2026-07-14)

### 1-6. 점검일자 확정 / 재확정 — 점검확정 목록 날짜 셀
- **함께 변경**: 항목 상태 확정 + 6단계 일정 자동계산(공휴일 회피). 재확정 시 완료 단계는 유지,
  미완료 단계만 재계산. 이미 점검이 시작된 항목이면 업무 체크리스트 마감일도 갱신
- **반영 화면**: 점검확정, 점검달력, (시작된 경우) 점검 상세 체크리스트

### 1-6b. 계획 항목 수정 — 점검확정 슬라이드 패널 저장 / 목록 인라인(담당 재배정·일괄 확정·날짜 비우기)
- **함께 변경 (2026-07-14 개편)**:
  - 예정일 변경·계획→확정: 재확정 경로(1-6)로 통일 — 자동 확정 + 6단계 재계산(공휴일 회피)
    + (시작된 점검) 업무 체크리스트 마감일·inspections 시작일 동기화(다일 점검 종료일 역전 시 함께 이동).
    1단계(점검일) 완료된 점검은 날짜 변경 거부(점검 상세에서만), 완료·취소 항목은 일정 변경 거부
  - 상태: 계획↔확정만 허용(확정 해제는 미시작만, 6단계 초기화). 완료·취소 직접 변경 불가

  - 담당직원: 점검확정에서 편집 불가(드롭다운 제거, 2026-07-14) — 항목 단위 배정이 고객 기본 담당으로
    역전파되지 않아 혼동을 낳으므로 담당 변경은 고객관리(1-2)에서만. 화면에는 읽기 전용 + "(퇴사)" 경고 표시
  - 메모(notes): inspection_plan_items에만 저장, 전파 없음
- **부가**: 활동 로그(plan_item_updated, 변경 전→후)
- **반영 화면**: 점검확정, 점검현황 모니터링, 점검업무, 점검달력
- **구현**: inspection-plans/actions.ts `updatePlanItemAction`
