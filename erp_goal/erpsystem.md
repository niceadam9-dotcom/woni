# 승진 유지보수 관리시스템 목표 문서

## 프로젝트 개요

소방업체 사내 유지보수 관리 업무 효율화를 위한 웹 기반 ERP(전사적 자원관리) 시스템 구축.
직원 로그인, 업체소방점검 일 기준으로 1차점검 및 관계인 통보(3일내), 2차 관계인 협의 통보 보강(1차후 3일), 소방서 제출문서(9일) 
점검 후 3일 그리고 3일 후 1차점검 보고서 제출, 3일 후 2차 협의 보고서 제출, 9일 후 소방서 제출문서 완료 확인(점검 후 15일내 담당자 통보) 
공사완료를 10일  공사완료 (10일, 공휴일 포함 계산)

 전자 기안서 작성 및 결재, 휴가 신청 기능을 핵심으로 한다.


---

## 핵심 기능 목표

### 1. 사용자 인증 및 권한 관리
- 직원 계정 로그인 / 로그아웃
- 역할(Role) 기반 권한 구분: 일반 직원 / 팀장 / 관리자
- 로그인 기록 및 접속 이력 저장

### 2. 전자 기안서 (문서 기안 및 결재)
- 기안서 작성: 제목, 내용, 첨부파일, 결재선 지정
- 결재선 설정: 1차 결재자 → 2차 결재자 → 최종 승인자
- 결재 상태 관리: 대기 / 승인 / 반려 / 회수
- 결재자 알림: 결재 요청 시 해당 결재자에게 알림
- 결재 이력 조회: 기안자 및 결재자 모두 진행 상황 확인 가능
- 반려 시 사유 입력 및 기안자에게 통보

### 3. 휴가 신청 시스템
- 휴가 신청서 작성: 휴가 종류(연차/반차/병가 등), 기간, 사유 입력
- 팀장 승인 → 관리자 최종 승인 워크플로우
- 연차 잔여일수 자동 계산 및 표시
- 휴가 신청 상태 조회: 대기 / 승인 / 반려
- 팀 단위 휴가 현황 캘린더 뷰

### 4. 기록 및 이력 관리
- 모든 기안서 / 휴가 신청 생성·수정·처리 이력 저장
- 관리자 대시보드: 전체 문서 현황, 미결재 건수 한눈에 확인
- 직원별 활동 로그 조회

## 시스템 요구사항

| 항목 | 내용 |
|------|------|
| 플랫폼 | 웹 기반 (브라우저 접속) |
| 사용자 | 사내 임직원 전체 |
| 인증 방식 | 아이디 / 비밀번호 (추후 SSO 확장 고려) |
| 데이터 저장 | 서버 DB (문서, 결재이력, 사용자 정보) |
| 접근 권한 | 역할 기반 접근 제어 (RBAC) |

## 비기능 요구사항 
1.**사용자 수** 동시 접속 최대 16명 기준 (확장기능 설계)
2.** 일 단위 DB 백업
---

## 화면 구성 (주요 페이지)

1. **로그인 페이지** — 계정 인증
2. **메인 대시보드** — 미결재 문서, 휴가 현황, 공지사항 요약
3. **기안서 작성** — 양식 선택 → 내용 작성 → 결재선 지정 → 상신
4. **결재함** — 내가 받은 결재 목록, 승인/반려 처리
5. **문서함** — 내가 기안한 문서 목록 및 상태 확인
6. **휴가 신청** — 휴가 신청서 작성 및 잔여 연차 확인
7. **휴가 현황** — 팀 캘린더, 승인된 휴가 일정
8. **관리자 페이지** — 직원 관리, 전체 문서/휴가 현황

## 개발 우선순위

1. 사용자 인증 시스템 (로그인/권한)
2. 기안서 작성 및 결재 워크플로우
3. 휴가 신청 및 승인 워크플로우
4. 대시보드 및 현황 조회
5. 관리자 기능
