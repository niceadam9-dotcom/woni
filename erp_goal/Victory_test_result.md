## 2. 🔴 크리티컬 버그 (배포 차단 수준)

### BUG-1. 고객관리 목록(/customers) 500 서버 에러 — 페이지 완전 다운

- **증상**: 로그인 상태에서 `/customers` 접근 시 "This page couldn't load. A server error occurred." (ERROR digest `3739023768`, `1730750776`)
- **원인**: 서버 컴포넌트 `erp/src/app/(dashboard)/customers/page.tsx` **229 / 265 / 292행**이 클라이언트 컴포넌트 `InlineCustomerFieldClient`에 `renderDisplay={v => <span>...</span>}` **함수를 prop으로 전달** → Next.js RSC 직렬화 불가.
  ```
  Error: Functions cannot be passed directly to Client Components unless you
  explicitly expose it by marking it with "use server".
  {customerId: ..., field: ..., value: ..., renderDisplay: function renderDisplay}
  ```
- **특징**: `tsc --noEmit`·`next build` 모두 통과 — **런타임에서만 발생**. 개발 중 이 페이지를 열어보지 않으면 잡히지 않음.
- **파급**: 고객관리 목록 의존 기능 전체 사용 불가 → **B-1~B-9(통합검색·인라인수정·필터칩·비활성전환·삭제), I-1~I-2(일반관리 점검일 등록), J-1(목록 인라인 담당자 변경)** 이 모두 BLOCKED.
- **수정 방향**: `renderDisplay` 함수 전달 제거 — 렌더링 분기를 클라이언트 컴포넌트 내부로 이동하거나(`displayVariant: 'bold' | 'badge'` 등 직렬화 가능한 값 전달), 표시용 JSX를 `children`으로 서버에서 렌더해 전달.
- **증적**: `shots/p8-customers-t4.png`, 서버 로그(백그라운드 태스크 출력)

### BUG-2. 6단계 마감일 이원화 — 확정일 변경이 업무체크리스트에 반영 안 됨

- **증상**: 동일 점검 건에 대해 서로 다른 두 벌의 6단계 날짜가 존재.
  | 소스 | 기준일 | 실측 (T1) |
  |------|--------|-----------|
  | `inspection_plan_items.step1~6_date` (confirmPlanItemStageOneAction) | **scheduled_date(확정일) 06-30** | step1=06-30, step4=07-22, step5=**08-01**, step6=08-14 |
  | `inspection_steps` (DB 트리거, 점검 시작 시 생성) | **use_approval_date(사용승인일) 06-30** | step1=07-01, step4=07-23, step5=**08-02**, step6=08-14 |
- **문제**: 슬라이드 패널·점검 상세의 6단계 업무체크리스트는 **트리거 값(inspection_steps)** 을 표시 → 관리자가 확정일을 변경해도 **업무체크리스트 마감일은 그대로**. Victory9 명세 "기준일 use_approval_date → scheduled_date(1단계 점검일자확정일)로 변경"이 **트리거(migration 019/036)에 미적용**.
- **수정 방향**: 점검 시작 시 inspection_steps.due_date를 plan_item의 step1~6_date로 동기화하거나, 트리거 기준일을 scheduled_date로 변경.
- **증적**: `shots/p6d-step-buttons.png` (패널 마감일) vs DB `inspection_plan_items` 조회값

### BUG-3. 공휴일 데이터 오류 — 제헌절이 공휴일로 등록됨

- **증상**: `holidays` 테이블에 `2026-07-17 제헌절 (is_national=true)` 존재. 제헌절은 **2008년부터 법정공휴일이 아님**.
- **파급**: 영업일 계산에서 7/17이 제외되어 **2·3·4·6단계 마감일이 실제보다 하루 늦게 산출** (예: 4단계 07-21 → 07-22). 소방시설법 법정 제출기한을 넘길 실무 리스크.
- **수정 방향**: `/api/cron/sync-holidays`가 공공데이터 API 응답에서 '국경일'과 '공휴일'을 구분하도록 수정 + 기존 잘못된 행 삭제.
- **참고**: 계산 엔진 자체는 공휴일 테이블 기준으로 전 단계 일관·정확함을 확인(D-4 PASS).
