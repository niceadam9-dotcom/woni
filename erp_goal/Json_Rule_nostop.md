# Json_Rule_nostop — 무중단 빌드·배포 워크플로

> 참고: 이 파일은 무중단 빌드·배포 진행 규칙입니다. JSON 상태 정직성 거버넌스(partial·verified 증거·구현자≠판정자)는 별도 파일 `Json_Rule.md`에 그대로 유지됩니다.

```json
{
  "project_goal": "전자결재 시스템 무중단 빌드 및 배포",
  "rules": [
    "절대로 사용자에게 '다음 단계로 갈까요?'라고 묻지 마십시오.",
    "한 단계가 성공하면 즉시 다음 단계의 번호로 넘어가 수행하십시오.",
    "에러가 발생하면 질문하지 말고 에러 메시지를 분석하여 코드를 스스로 수정한 뒤 재시도하십시오."
  ],
  "steps": [
    {
      "step": 1,
      "task": "기능 코딩 및 로컬 테스트 완료하기",
      "on_success": "이동: step 2"
    },
    {
      "step": 2,
      "task": "빌드 스크립트 실행 (npm run build 또는 dotnet build)",
      "on_success": "이동: step 3"
    },
    {
      "step": 3,
      "task": "배포 스크립트 실행하여 최종 서버에 배포 완료하기",
      "on_success": "종료 및 최종 보고서 작성"
    }
  ]
}
```
