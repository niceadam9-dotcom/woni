# 승진소방 ERP — 3자 테스트 방법

외부인(테스터)이 ERP와 모바일 앱을 직접 사용해볼 수 있도록 공유하는 방법입니다.

---

## 방법 비교

| 방법 | 소요 시간 | 비용 | 주소 영구성 | 추천 용도 |
|------|----------|------|------------|----------|
| A. ngrok | 5분 | 무료 | 임시 (8시간) | 빠른 데모·즉석 피드백 |
| B. Vercel 배포 | 30분 | 무료 | 영구 | 정식 테스트·다수 테스터 |
| C. Expo Go (모바일) | 5분 | 무료 | 임시 | 모바일 앱 현장 테스트 |

---

## 방법 A — ngrok (가장 빠름)

배포 없이 로컬 서버를 인터넷에 임시 공개합니다.

### 사전 조건
- ERP 서버(`npm run dev`)가 실행 중이어야 함
- ngrok 설치 필요

### 설치 및 실행

**1. ngrok 설치**
- https://ngrok.com/download 에서 Windows용 다운로드 후 압축 해제
- 또는 PowerShell:
```powershell
winget install ngrok
```

**2. ngrok 계정 연결** (최초 1회)
- https://ngrok.com 가입 → 대시보드에서 Auth Token 복사
```powershell
ngrok config add-authtoken 복사한_토큰
```

**3. ERP 서버 실행**
```powershell
cd F:\AI\ERP\erp
npm run dev
```

**4. 터널 개설 (새 터미널)**
```powershell
ngrok http 3000
```

**5. 테스터에게 전달**
```
출력 예시:
Forwarding  https://abc123.ngrok-free.app → http://localhost:3000

→ https://abc123.ngrok-free.app 주소를 테스터에게 공유
```

### 테스터 접속 정보 전달 예시

```
[승진소방 ERP 테스트 안내]

접속 URL: https://abc123.ngrok-free.app

테스트 계정:
- 직원:   park.dev@erp-test.com   / Employee1!
- 매니저: dev.manager@erp-test.com / Manager1!
- 관리자: admin@erp-test.com       / Admin1234!

※ 이 주소는 8시간 후 만료됩니다.
※ 데이터를 자유롭게 입력·수정해도 됩니다.
```

### 주의사항
- 세션당 8시간 제한 (무료 플랜)
- 테스트 종료 후 터미널에서 `Ctrl+C`로 ngrok 종료
- 매번 주소가 바뀜 (고정 주소는 ngrok 유료 플랜 월 $8)

## 방법 B — Vercel 배포 (권장)

영구 URL로 언제든 접속 가능한 정식 테스트 환경을 만듭니다.

### 1. GitHub에 코드 업로드

```powershell
cd F:\AI\ERP
git init                          # 아직 git 저장소가 없다면
git add erp/
git commit -m "ERP 배포"
# GitHub에서 새 저장소 생성 후:
git remote add origin https://github.com/계정명/저장소명.git
git push -u origin main
```

### 2. Vercel 배포

1. https://vercel.com 가입 (GitHub 계정 연동)
2. **New Project** → 위에서 만든 저장소 선택
3. **Root Directory**: `erp` 입력
4. **Environment Variables** 탭에서 아래 입력:

| 변수명 | 값 |
|--------|----|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://ryuozdhnilfjlahorizh.supabase.co` |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `eyJhbGci...` (erp/.env.local에서 복사) |
| `SUPABASE_SERVICE_ROLE_KEY` | `eyJhbGci...` (erp/.env.local에서 복사) |
| `ANTHROPIC_API_KEY` | 실제 Anthropic API 키 |

5. **Deploy** 클릭
6. 완료 후 `https://your-app.vercel.app` 주소 발급

### 3. 테스터에게 전달

```
[승진소방 ERP 테스트 안내]

접속 URL: https://your-app.vercel.app

테스트 계정:
- 직원:   park.dev@erp-test.com   / Employee1!
- 매니저: dev.manager@erp-test.com / Manager1!
- 관리자: admin@erp-test.com       / Admin1234!

모바일 앱: https://your-mobile-app.vercel.app (별도 배포 시)

테스트 항목:
1. 로그인
2. 대시보드 확인
3. 고객 등록 및 검색
4. 점검 배정 및 단계 완료 처리
5. 불량 등록 (AI 분류 포함)
```

### 4. 코드 수정 시 자동 재배포

```powershell
git add .
git commit -m "수정 내용"
git push
# → Vercel이 자동으로 재빌드·배포
```

## 방법 C — Expo Go (모바일 앱 현장 테스트)

테스터 스마트폰에서 모바일 현장점검 앱을 바로 실행합니다.

### 사전 조건
- 테스터와 같은 Wi-Fi 네트워크에 연결되어 있어야 함
- 테스터 폰에 **Expo Go** 앱 설치 필요
  - Android: Google Play → "Expo Go" 검색
  - iOS: App Store → "Expo Go" 검색

### 실행

```powershell
cd F:\AI\ERP\mobile
npx expo start
```

터미널에 QR 코드가 표시됩니다.

```
테스터 안내:
1. 폰에서 Expo Go 앱 실행
2. "Scan QR Code" 탭 → 화면의 QR 코드 스캔
3. 앱 자동 실행

로그인 계정:
- 직원: park.dev@erp-test.com / Employee1!
```

### 테스트 항목 (모바일)

```
1. 로그인
2. 점검 목록 확인
3. 점검 시작 버튼 클릭
4. 불량 등록 → AI 분류 버튼 테스트
5. 사진 첨부
6. 현장 서류 탭 확인
7. 내 정보 → 로그아웃
```

### 주의사항
- 서버가 종료되면 앱도 중단됨
- 다른 Wi-Fi 환경에서는 접속 불가
- 원거리 테스터에게는 방법 B(Vercel) 사용 후 모바일 웹브라우저로 접속 안내

테스터에게 아래 항목을 확인해달라고 요청합니다.

[ ] 대시보드 데이터 표시

[ ] 점검계획 등록 및 자동 생성
[ ] 휴가 신청 및 승인
[ ] 전자결재 기안·승인
[ ] 게시판 글 작성·댓글
[ ] 회의록 작성

```

[ ] 내 점검 목록 표시

```
