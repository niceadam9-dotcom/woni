## 1. 시스템 개요

승진소방 ERP는 소방안전관리 업무 전반을 디지털로 통합 관리하는 시스템입니다.

| 구분 | 내용 |
|------|------|
| 웹 주소 | `http://localhost:3000` (또는 배포된 도메인) |
| 모바일 앱 | `http://localhost:8081` (웹 브라우저) 또는 Expo Go 앱 |
| 지원 브라우저 | Chrome, Edge, Safari (최신 버전) |
| 권장 화면 해상도 | 1280×800 이상 |

### 시스템 구성

```
승진소방 ERP (웹)
├── 소방안전관리 — 고객·점검·보고서·이행계획·정산
├── 업무관리      — 업무지시·일지·차량
├── 영업관리      — 견적·수주·거래처
├── 인사/회계     — 휴가·급여·전표·재무
└── 전자결재      — 기안·결재·문서 보관

모바일 앱 (현장점검)
└── 점검계획 확인 → 점검 시작 → 불량 등록 → 서류 작성
```

## 2. 로그인 및 권한

### 로그인

1. 브라우저에서 ERP 주소 접속
2. 이메일·비밀번호 입력 후 **로그인** 클릭
3. 로그인 성공 시 대시보드로 이동

### 권한 체계

| 권한 | 대상 | 접근 범위 |
|------|------|----------|
| **employee** (일반 직원) | 현장 점검 담당자 | 소방안전관리(업무 전반: 고객·건물·점검표·계획·보고서·이행계획)·My Page·전자결재(기안)·업무관리 |
| **manager** (매니저) | 팀장급 | 직원 권한 + 결재·휴가승인·급여·영업·재고·회계 + 소방 담당배정·삭제·정산·세금계산서 |
| **admin** (관리자) | 시스템 관리자 | 전체 + 직원 관리·부서·공휴일·시스템 설정 |

> **2026-07-08 변경**: 소방안전관리 모듈은 일반 직원도 매니저와 동등하게 업무를 수행합니다 (고객·건물 등록/수정, 점검계획 생성·확정, 보고서 제출현황, 이행계획서).
> 단 **돈(정산현황·세금계산서), 삭제(고객·점검), 담당자 배정(지역별 배정·이관)** 은 매니저 이상 전용입니다.
> 점검업무·점검달력은 일반 직원도 전체 조회가 가능하며, 첫 화면 기본 필터만 '내 담당'입니다.

> **테스트 계정**
> - 관리자: `admin@erp-test.com` / `Admin1234!`
> - 매니저: `dev.manager@erp-test.com` / `Manager1!`
> - 직원: `park.dev@erp-test.com` / `Employee1!`

### 주요 구성

## 4. My Page — 개인 업무

개인이 사용하는 기능 모음입니다. 모든 직원이 접근할 수 있습니다.

### 4-1. 일정 관리 `/my/schedules`

- 개인 캘린더에 일정 등록·수정·삭제
- 제목·날짜·시간·메모 입력
- 월/주/일 뷰 전환

### 4-2. ToDo 목록 `/my/todos`

- 할 일 목록 관리
- 완료 체크 시 취소선 표시
- 우선순위·마감일 설정 가능

### 4-3. 주소록 `/my/address-book`

- 개인 연락처 등록 및 검색
- 이름·전화번호·이메일·소속 저장

### 4-4. 쪽지함 `/my/messages`

- 직원 간 내부 메시지 송수신
- 받은 쪽지 / 보낸 쪽지 구분
- 읽음 여부 표시

### 4-5. 노트 `/my/notes`

- 자유 형식 개인 메모
- 마크다운 형식 지원

### 4-6. 녹음 메모장 `/my/voice-memos`

- 브라우저 마이크로 음성 녹음 후 저장
- 녹음 파일 목록 재생

### 4-7. 결재서명 `/my/signature`

- 전자결재 서명 이미지 등록
- 서명 패드에서 직접 서명 또는 이미지 업로드

## 5. 전자결재

문서 기안·결재·보관을 처리합니다.

### 5-1. 문서함 `/documents`

모든 직원이 기안한 문서를 조회하고 새 문서를 작성합니다.

**새 문서 작성**
1. 우상단 **+ 문서 작성** 클릭
2. 문서 유형 선택 (보고서, 품의서, 휴가신청 등)
3. 내용 작성 후 결재라인 지정
4. **기안** 클릭 → 결재자에게 알림 발송

**문서 상태**
| 상태 | 의미 |
|------|------|
| 초안 | 아직 기안하지 않은 상태 |
| 결재 대기 | 결재자 검토 중 |
| 승인 | 결재 완료 |
| 반려 | 수정 후 재기안 필요 |
| 회수 | 기안자가 취소 |

### 5-2. 결재함 `/approvals` (매니저·관리자)

대기 중인 결재 문서를 처리합니다.

1. 목록에서 문서 클릭 → 내용 검토
2. **승인** 또는 **반려** 선택
3. 반려 시 사유 입력 필수

## 6. 인사 / 휴가

### 6-1. 휴가 신청 `/leaves`

1. **+ 휴가 신청** 클릭
2. 휴가 유형 선택 (연차, 반차, 병가, 경조사 등)
3. 날짜 범위 선택 및 사유 입력
4. **신청** 클릭 → 매니저에게 승인 요청

**잔여 연차** — 화면 상단에 사용 가능한 연차 일수 표시

### 6-2. 팀 휴가 캘린더 `/leaves/calendar`

팀원 전체의 휴가 현황을 캘린더로 확인합니다.

### 6-3. 휴가 승인 `/leaves/manage` (매니저·관리자)

1. 대기 중인 휴가 신청 목록 확인
2. 신청 클릭 후 **승인** 또는 **반려**
3. 승인 시 신청자에게 알림 발송

### 6-4. 급여 등록 `/hr/payroll` (매니저·관리자)

- 직원별 월 급여 입력 (기본급·수당·공제)
- 급여 명세서 출력

### 6-5. 증명서 발급 `/hr/certificates` (매니저·관리자)

- 재직증명서·경력증명서 발급
- 직원 선택 후 **발급** 클릭 → PDF 다운로드

## 7. 소방안전관리

승진소방 ERP의 핵심 기능입니다. 고객 계약부터 점검 완료·보고서 제출·세금계산서 발행까지 전체 업무 흐름을 관리합니다.

### 업무 흐름 전체 개요

```
고객 등록 → 건물 등록 → 점검계획 등록
    ↓
점검 배정 → [현장 점검 (모바일)] → 점검 단계 완료
    ↓
보고서 제출 → 이행계획서 등록 → 이행완료 보고서
    ↓
정산현황 → 세금계산서 발행
```

---

### 7-1. 고객 관리 `/customers`

소방 점검 계약 고객을 등록하고 관리합니다.

#### 고객 목록

- 검색: 고객명·고객코드·주소로 검색
- 필터: 점검유형(종합/최초/기타), 활성/비활성
- 페이지당 50건 표시

#### 고객 등록

1. **+ 고객 등록** 클릭
2. 필수 정보 입력:
   - 고객명, 사업자등록번호
   - 계약일, 점검유형
   - 담당 직원 지정
   - 주소
3. **저장** 클릭

#### 점검유형별 연간 점검 횟수

| 유형 | 연간 횟수 |
|------|----------|
| 종합 | 연 2회 (1차·2차) |
| 최초 | 연 1회 |
| 기타 | 연 1회 |

#### 고객 상세 화면

- **기본정보** 탭 — 고객 정보 수정
- **연락처** 탭 — 담당자 연락처 관리
- **점검이력** 탭 — 과거 점검 목록 확인
- **담당직원 변경** — 담당자 재배정

---

### 7-2. 건물 관리 `/buildings`

고객사의 건물 정보를 관리합니다.

- 건물명·주소·용도·연면적·층수 등록
- 소방 시설 현황 입력
- 고객사에 건물 연결

---

### 7-3. 문의요청 `/inquiries`

고객 문의 및 요청사항을 접수·처리합니다.

1. **+ 문의 등록** 클릭
2. 고객 선택, 내용 작성, 담당자 지정
3. 처리 완료 시 상태를 **완료**로 변경

---

### 7-4. 점검표 관리 `/inspection-sheets`

현장에서 사용하는 점검 체크리스트 양식을 관리합니다.

- 점검표 항목 등록·수정·삭제
- 점검 유형별 기본 양식 제공
- 점검 수행 시 이 양식으로 기록

---

### 7-5. 점검 업무 `/inspections`

실제 점검 업무의 핵심입니다.

#### 점검 목록

| 필터 | 설명 |
|------|------|
| 연도 | 해당 연도의 점검만 표시 |
| 상태 | 예정/진행중/완료/기한초과 |
| 담당자 | 특정 직원의 점검만 표시 |

#### 점검 상태

| 상태 | 의미 |
|------|------|
| 예정 | 점검 시작 전 |
| 진행중 | 7단계 중 일부 완료 |
| 완료 | 모든 단계 완료 |
| 기한초과 | 마감일이 지난 미완료 단계 존재 |

#### 새 점검 배정 (매니저·관리자)

1. **점검 배정** 클릭
2. 고객 선택, 차수(1차/2차), 시작일, 담당 직원 지정
3. **저장** 시 자동으로 7단계 생성

#### 7단계 점검 프로세스

| 단계 | 이름 | 설명 |
|------|------|------|
| 1단계 | 관계인 통보 1차 | 점검 일정 사전 통보 |
| 2단계 | 관계인 협의 2차 보강 | 현장 협의 및 보완 |
| 3단계 | 보고서 작성 | 점검 결과 보고서 작성 |
| 4단계 | 소방서 제출 | 관할 소방서에 보고서 제출 |
| 5단계 | 공사완료 | 불량사항 조치 공사 완료 |
| 6단계 | 이해관계자 보고서 | 관계자 보고서 제출 |
| 7단계 | 이행완료 보고서 제출 | 이행 완료 최종 보고 |

각 단계마다 마감일 설정 가능. 단계 클릭 → **완료 처리** 클릭으로 진행.

#### 불량 등록

점검 중 발견한 불량을 등록합니다.

1. 점검 상세 화면에서 **+ 불량 등록** 클릭
2. 불량 항목명, 상세 내용, 심각도 선택
3. 사진 첨부 (선택)
4. **AI 분류** 버튼으로 Claude AI가 자동 분류

**심각도 기준**
| 심각도 | 기준 |
|--------|------|
| 경미 | 즉시 위험 없음, 정기 점검 시 조치 |
| 보통 | 조속한 조치 필요, 잠재적 위험 |
| 중대 | 즉각 조치 필요, 화재 시 인명피해 위험 |

### 7-6. 점검 달력 `/inspections/calendar`

이번 달 점검 일정을 달력 형식으로 확인합니다.

- 날짜별 점검 건수 표시
- 날짜 클릭 → 해당일 점검 목록

---

### 7-7. 점검계획 등록 `/inspection-plans`

연간·월간 점검 계획을 수립합니다.

#### 계획 등록

1. 해당 연월 선택
2. **+ 계획 추가** 또는 **자동 생성** 클릭

#### 자동 생성 기능

- 활성 고객 전체를 대상으로 점검 계획 자동 생성
- 종합 고객: 1차(상반기)·2차(하반기) 자동 배분
- 담당 직원별 균등 배분

#### 계획 항목 관리

- 일정 조정: 항목 클릭 → 날짜 변경
- 담당자 변경: 드롭다운에서 직원 선택
- 상태: 예정 → 확정 → 완료

---

### 7-8. 점검현황 모니터링 `/inspection-plans/monitor`

전체 점검 진행 현황을 한눈에 봅니다.

- 고객별 1차·2차 완료 현황
- 지연·미진행 건 강조 표시
- 담당자별 업무량 집계

---

### 7-9. 보고서 제출현황 `/inspection-reports/status`

각 점검의 보고서 제출 상태를 관리합니다.

| 상태 | 의미 |
|------|------|
| 미제출 | 보고서 아직 작성 안 됨 |
| 작성중 | 보고서 초안 작성 중 |
| 제출완료 | 소방서 제출 완료 |

---

### 7-10. 이행계획서 등록 `/action-plans`

불량사항에 대한 이행계획서를 작성합니다.

1. **+ 이행계획 등록** 클릭
2. 해당 점검 선택
3. 불량항목별 조치 계획·담당자·기한 입력
4. **제출** 클릭

---

### 7-11. 이행계획 제출현황 `/action-plans/status`

이행계획서 제출 현황 및 완료 상태를 모니터링합니다.

---

### 7-12. 정산현황 `/billing/status` (매니저·관리자)

## 14. 모바일 앱 — 현장점검

현장 점검 담당자가 스마트폰·태블릿에서 사용합니다.

### 접속 방법

| 방법 | 주소 |
|------|------|
| 웹 브라우저 | `http://localhost:8081` |
| Expo Go 앱 | QR 코드 스캔 (expo start 후 제공) |

### 화면 구성

```
하단 탭 메뉴
├── 📋 점검 목록 — 내 담당 점검 계획 목록
├── 📄 서류       — 현장 서류 작성
└── 👤 내 정보   — 프로필·로그아웃
```

### 점검 업무 흐름

#### 1. 점검 목록 확인

- 오늘 점검: 상단에 강조 표시
- 예정 점검: 날짜순 정렬

#### 2. 점검 시작

1. 목록에서 고객 탭
2. **점검 시작** 버튼 클릭
3. 자동으로 7단계 생성 및 점검 번호 부여

#### 3. 불량 등록

1. **+ 불량 등록** 클릭
2. 불량 항목명 입력
3. **AI 분류** 버튼 → Claude AI가 자동으로 심각도 분류
4. 📷 사진 촬영 또는 파일 선택으로 증빙 첨부
5. **저장**

> 🎙 네이티브 앱(Expo Go)에서는 음성 녹음 후 텍스트 변환 → AI 분류도 가능합니다.

#### 4. 단계 완료 처리

각 점검 단계 우측의 **완료** 버튼 클릭 → 완료 시간 자동 기록

#### 5. 현장 서류 작성

서류 탭에서 다음 서류를 작성합니다:

| 서류 | 설명 |
|------|------|
| 소방계획서 | 해당 건물의 소방 대응 계획 |
| 업무수행기록표 | 점검 업무 수행 내역 기록 |
| 자체점검기록부 | 자체 점검 결과 기록 |
| 자위소방대·교육훈련 | 소방 교육·훈련 기록 |
| 화재/비화재보 기록부 | 화재경보 발생 기록 |

## 부록: 자주 묻는 질문 (FAQ)

**Q. 비밀번호를 잊어버렸습니다.**
> 관리자(admin)에게 문의하여 비밀번호 재설정을 요청하세요.

**Q. 점검 단계를 완료 처리했는데 취소하고 싶습니다.**
> 현재 버전에서는 완료 처리 취소가 지원되지 않습니다. 관리자에게 문의하세요.

**Q. 고객을 삭제하고 싶습니다.**
> 고객 삭제 대신 **비활성화** 처리를 권장합니다. 삭제 시 연결된 점검 이력도 모두 삭제됩니다.

**Q. AI 불량 분류가 작동하지 않습니다.**
> `.env.local`에 `ANTHROPIC_API_KEY`가 설정되어 있는지 확인하세요.

**Q. 모바일 앱에서 음성 녹음이 되지 않습니다.**
> 웹 브라우저에서는 음성 녹음이 지원되지 않습니다. Expo Go 네이티브 앱을 사용하세요.
