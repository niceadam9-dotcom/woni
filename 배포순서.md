## ④ ERP(Next.js) 배포

아래 세 가지 옵션 중 선택합니다.

---

### 옵션 A — Vercel (권장, 무료)

Next.js 공식 플랫폼으로 가장 쉽게 배포할 수 있습니다.

1. https://vercel.com 가입 (GitHub 계정으로 로그인)
2. **New Project** 클릭 → GitHub 저장소 연결
3. **Root Directory**: `erp` 입력
4. **Environment Variables** 탭에서 아래 5개 입력:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `ANTHROPIC_API_KEY`
   - (선택) `RESEND_API_KEY`, `NEXT_PUBLIC_VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT`
5. **Deploy** 클릭
6. → `https://your-app.vercel.app` 자동 발급

**커스텀 도메인 연결** (선택)
- Vercel 프로젝트 → Settings → Domains → 도메인 입력

| 장점 | 단점 |
|------|------|
| 무료, 자동 HTTPS | 서버리스 함수 실행시간 제한 (무료 60초) |
| GitHub push 시 자동 재배포 | |

---

### 옵션 B — Railway (월 ~$5)

Docker 없이 Git만으로 배포됩니다.

1. https://railway.app 가입
2. **New Project** → **Deploy from GitHub**
3. 저장소 선택, Root Directory: `erp`
4. 환경변수 설정 (Variables 탭)
5. 자동 빌드 및 배포
6. Settings → Domains → 커스텀 도메인 연결 가능

---

### 옵션 C — VPS + Docker (월 $5~, 완전 제어)

`next.config.ts`에 `output: 'standalone'`이 설정되어 있어 경량 Docker 이미지를 만들 수 있습니다.

**Dockerfile** (`F:\AI\ERP\erp\Dockerfile` 생성)
```dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public
EXPOSE 3000
CMD ["node", "server.js"]
```

**VPS에서 실행**
```bash
docker build -t sjfire-erp .
docker run -d -p 3000:3000 \
  -e NEXT_PUBLIC_SUPABASE_URL=... \
  -e NEXT_PUBLIC_SUPABASE_ANON_KEY=... \
  -e SUPABASE_SERVICE_ROLE_KEY=... \
  -e ANTHROPIC_API_KEY=... \
  --name sjfire-erp \
  sjfire-erp
```

nginx로 리버스 프록시 + Let's Encrypt로 HTTPS 설정 권장.

### ⑤-2. 웹 앱으로 배포 (Vercel/Netlify)

- **Netlify**: `dist/` 폴더를 드래그 앤 드롭 배포

## ⑥ 서비스 시작 — 관리자 계정 초기 설정

ERP 배포 완료 후 아래 순서로 초기 설정을 진행합니다.

1. 배포된 ERP URL 접속
2. **관리자 계정**으로 로그인
   ```
   이메일: admin@erp-test.com
   비밀번호: Admin1234!
   ```
3. `/admin/users` → 실제 직원 계정 생성
4. `/admin/departments` → 부서 등록
5. `/admin/holidays` → 해당 연도 공휴일 등록
6. `/company` → 회사 기본 정보 입력
7. `/customers` → 고객 등록 시작

> **보안 주의**: 서비스 오픈 전 테스트 계정(`admin@erp-test.com` 등)의 비밀번호를 반드시 변경하거나 삭제하세요.
